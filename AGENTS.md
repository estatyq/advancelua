# AGENTS.md — Архитектура helper_core.lua

> Universal Game Helper Core for SAMP (MoonLoader / Lua)
> Файл: `helper_core.lua` — ядро и каркас платформы-помощника для сервера Advance RP.
> Версия скрипта: v0.9 (28.06.2026). Меню: **F11** или команда `/helper`.

## 0. КРИТИЧЕСКИЕ ПРАВИЛА РАБОТЫ С ФАЙЛОМ

> **ВНИМАНИЕ! Файл `helper_core.lua` хранится в кодировке CP1251.**

1. **Чтение файла** — только через PowerShell с явным указанием кодировки:
   ```powershell
   $b = [System.IO.File]::ReadAllBytes("c:/Users/maxhy/OneDrive/Desktop/advance sapm/helper_core.lua")
   $cp = [System.Text.Encoding]::GetEncoding(1251)
   $text = $cp.GetString($b)
   ```
2. **Запись/правки файла** — только через PowerShell:
   ```powershell
   $cp = [System.Text.Encoding]::GetEncoding(1251)
   [System.IO.File]::WriteAllBytes("c:/Users/maxhy/OneDrive/Desktop/advance sapm/helper_core.lua", $cp.GetBytes($text))
   ```
   **НЕ использовать `edit` tool** — он ломает кодировку (пишет UTF-8, скрипт перестаёт загружаться).
3. **После любых правок** — скопировать файл в рабочую папку MoonLoader:
   ```
   c:/Users/maxhy/OneDrive/Desktop/advance sapm/helper_core.lua
   → E:\SAMP\moonloader\helper_core.lua
   ```
4. **Перезагрузка скрипта в игре** — **Ctrl+R** (перезагрузка MoonLoader).
   Команды `/ts` **не существует** — не предлагать её.
5. **JSON-файлы настроек** хранятся в `%LOCALAPPDATA%` (`getFolderPath(0x1C)`):
   - `helper_rp_settings.json` — отыгровки РП-движка
   - `helper_blacklist.json` — чёрный список СМИ
   - `helper_ad_history.json` — история объявлений
6. **JSON-файлы конфигурации** хранятся в `getWorkingDirectory()/config/`:
   - `helper_db.json` — база контактов (обзвон)
   - `helper_mm_rules.json` — правила сокращений MM Editor
   - `helper_settings.json` — общие настройки скрипта

## 3. СОБЫТИЯ SAMP (sampev.*)

Все обработчики событий определены через `sampev` (`lib.samp.events`).

### 3.1 `sampev.onSendChat(message)` — строка ~5503
Перехват исходящих сообщений/команд. Если включён модуль `auto_rp`, добавляет
`/me`-отыгровки перед серверными командами:

| Команда | Условие | Отыгровка |
|---|---|---|
| `/call <номер>` (или `/c`) | `rp_phone_enabled` | `/me достал мобильный телефон и набрал номер <номер>` |
| `/h` (или `/hangup`) | `rp_phone_enabled` | `/me закрыл телефон и убрал его в карман` |
| `/mask` | `rp_mask_enabled` | `/me надел на лицо маску и скрыл свое лицо` |
| `/healme` | `rp_heal_enabled` | `/me достал аптечку, открыл ее и применил` |
| `/drugs` | `rp_heal_enabled` | `/me достал шприц и сделал укол` |
| `/e` (двигатель) | `rp_weapons_enabled` | `/me завел двигатель` |

Возвращает `true` — сообщение проходит на сервер.

### 3.2 `sampev.onSetPlayerTime(hour, minute)` — строка ~5553
Если `time_locked[0] == true` → возвращает `false` (блокирует серверную синхронизацию времени).
Локальное время управляется потоком `environmentWorker` (запись в память GTA).

### 3.3 `sampev.onSetWeather(weatherId)` — строка ~5559
Если `weather_locked[0] == true` → возвращает `false` (блокирует серверную синхронизацию погоды).
Локальная погода управляется потоком `environmentWorker`.

### 3.4 `sampev.onServerMessage(color, text)` — строка ~5566
Обработка серверных сообщений (текст в CP1251):

**Сбор контактов (модуль `autocall_db`):**
- Парсит `Отправил <ник>[<id>] (тел. <номер>)` → сохраняет в `player_db`.
- Если отправитель = свой ник и активен `auto_ad` → планирует следующую подачу объявления.

**Авто-отыгровки (модуль `auto_rp`):**
| Событие (поиск в тексте) | Условие | Отыгровка |
|---|---|---|
| «вам звонит» / «входящий вызов» / «входящий звонок» | `rp_phone_enabled` | `/me достал мобильный телефон из кармана` → `/me посмотрел на экран` → `/do Телефон в руке` |
| «звонок завершен» / «вы сбросили» / «разговор окончен» / «собеседник сбросил» | `rp_phone_enabled` | `/me нажал кнопку сброса` → `/me убрал мобильный телефон в карман` |
| «sms:» / «смс:» / «вам пришло сообщение» / «новое сообщение» | `rp_phone_enabled` | `/me достал телефон` → `/me прочитал сообщение` → `/me убрал телефон` |
| «вы арестованы» / «посажен в кпз» / «отправлен в камеру» | `rp_heal_enabled` | `/me опустил голову` → `/do Выражение лица — подавленное` |
| «вы вылечены» / «здоровье восстановлено» | `rp_heal_enabled` | `/me облегчённо выдохнул` → `/do Самочувствие улучшилось` |

**Эфир-статистика (если `efir_active`):**
- Подсчёт SMS (`efir_stats.sms`), звонков (`efir_stats.calls`), строк эфира (`efir_stats.lines`).
- При звонке и `auto_answer_enabled` → отправляет `/t <текст авто-ответа>`.

### 3.5 `sampev.onShowDialog(dialogId, style, title, button1, button2, text)` — строка ~5680
Перехват диалоговых окон:

1. **Чтение статистики (РП-движок):** если `rp_reading_stats` и заголовок содержит
   «Статистика» → сохраняет текст в `rp_stats_text`, закрывает диалог (`return false`).
2. **Поиск сотрудников (`/find`):** если заголовок содержит «Поиск» / «Сотрудники» →
   парсит список (ник, id, ранг, подразделение, телефон) в `find_list`, открывает GUI
   (`find_show[0] = true`), `return false`.
3. **MM Editor (модуль `mm_editor`):** если заголовок содержит «публикац» / «объявлен»
   или текст содержит «Текст:» → извлекает оригинальный текст, форматирует через
   `formatAdText()`, открывает окно AutoEdit (`ae_active[0] = true`), `return false`.

## 6. РП-ДВИЖОК

### 6.1 Структура данных

```lua
-- Данные игрока (строка ~3979)
user = {
    nick, fullName, name, family,   -- ник, ФИО, имя, фамилия
    rang, rangName, podr, podrNum,  -- ранг (число), должность, подразделение, номер фракции
    phone, id, isWork               -- телефон, ID игрока, на работе ли
}

-- Данные цели (строка ~3986, для /mmact)
userTarget = { id, nick, name }
```

### 6.2 Теги отыгровок

Теги определены в таблице `rp_tegs` (строка ~4011). Заменяются функцией `playRp`.

**Общие теги (`rp_tegs[1]`)** — доступны во всех главах:

| Тег | Значение |
|---|---|
| `<time>` | Текущее время `os.date('%X')` |
| `<date>` | Текущая дата `os.date('%d.%m.%Y')` |
| `<myFio>` | Полное ФИО игрока (`user.fullName`) |
| `<myName>` | Имя игрока (`user.name`) |
| `<myNick>` | Ник игрока (`user.nick`) |
| `<myRang>` | Должность (`user.rangName`) |
| `<myPodr>` | Подразделение (`user.podr`) |
| `<myId>` | ID игрока на сервере |
| `<myPhone>` | Телефон игрока (`user.phone`) |

**Теги цели (`rp_tegs[2]`)** — доступны только в главах 1 и 3 (не в главе 2):

| Тег | Значение |
|---|---|
| `<tFio>` | ФИО цели (`userTarget.name`) |
| `<tName>` | Имя цели (часть ника до `_`) |
| `<tNick>` | Ник цели (`userTarget.nick`) |
| `<tId>` | ID цели (`userTarget.id`) |

> Выбор тегов зависит от главы: `setCurTeg(chapter)` (строка ~4042).
> Глава 2 → только общие теги. Главы 1 и 3 → общие + теги цели.

### 6.3 Формат отыгровок

Отыгровка — многострочный текст. Каждая строка обрабатывается отдельно в `playRp`
(строка ~4278):

| Конструкция | Описание |
|---|---|
| `r:{вариант1}{вариант2}r` | **Рандом**: случайно выбирается один из вариантов в `{}`. Обрабатывается до построчного разбора. |
| `<N>` (например `<600>`) | **Пауза** N миллисекунд. `<0>` — пауза с ожиданием: выводит «Пауза. /mmnext — продолжить», ждёт `/mmnext`. |
| `#input:` | **Окно ввода**: открывает GUI с текстовым полем. Введённое значение подставляется через `{w}`. |
| `#list: {a}{b}{c}` | **Окно выбора**: открывает GUI со списком вариантов. Выбранный вариант подставляется через `{w}`. |
| `{w}` | **Подстановка**: заменяется на значение из последнего `#input:` или `#list:`. |
| `#close:` | **Закрыть окно** ввода/выбора. |
| Обычный текст | Отправляется в чат через `sampSendChat(line)` с задержкой 500 мс. |

### 6.4 Главы отыгровок

`rp_settings.set` — таблица с 3 главами (`[1]`, `[2]`, `[3]`):

| Глава | Индекс `setList` | Название | Описание |
|---|---|---|---|
| 1 | 0 | Общие | Вспомогательные отыгровки (закурить, паспорт, рация, заправка и т.д.). Предзаполнены по умолчанию. |
| 2 | 1 | Мои | Пустые — пользователь заполняет сам. Только общие теги. |
| 3 | 2 | Для цели | Интерактивные отыгровки с тегами цели (проверка документов, досмотр, обыск, задержание, посадка в машину и т.д.). |

> `initDefaultRp()` (строка ~4061) заполняет главу 1 (20 отыгровок) и главу 3

## 8. СМИ-ФУНКЦИИ (модуль `smi_tools`)

### 8.1 Анаграммы (`/anag`)

- **Синтаксис:** `/anag [1-3]`
- **Типы:** 1 = слова (радио), 2 = телефон, 3 = газета.
- **Слова** (строка ~4429): по 10 слов на каждый тип.
- **Логика:** (`startAnagram`, строка ~4456) — выбирает случайное слово, перемешивает
  буквы (`createAnagram`), вставляет в чат с разделителем `.`.
- **Префикс:** если подразделение содержит «радио» → `/t`, если «центр» → `/u`.

### 8.2 Эфир (`/mmefir`, `/endefir`, `/efirstats`)

- **Начало эфира** (`cmdEfir`, строка ~4555): `efir_active = true`, запускает `playRp`
  с текстом: `/me надел наушники и настроил микрофон` → `/do Микрофон включён` → `/efir`.
- **Конец эфира** (`cmdEndEfir`, строка ~4565): `playRp` с текстом:
  `/me снял наушники` → `/do Наушники сняты` → `/efir`. `efir_active = false`.
- **Статистика** (`cmdEfirStats`, строка ~4637): время эфира, кол-во SMS, звонков, строк.
  Подсчёт ведётся в `sampev.onServerMessage` (строка ~5658).
- **Структура:** `efir_stats = { sms, calls, lines, start_time }`.

### 8.3 Авто-ответ (`/autoans`)

- **Синтаксис:** `/autoans [текст]` — включить/выключить.
- **Логика** (`cmdAutoAnswer`, строка ~4653): при звонке во время эфира
  (`efir_active` + `auto_answer_enabled`) отправляет `/t <текст>` с задержкой 500 мс.
- Текст хранится в `auto_answer_text` (imgui char[256]).

### 8.4 Поиск сотрудников (`/find` + GUI)

- **Команда** (`cmdFind`, строка ~4580): отправляет `/find` на сервер.
- **Перехват** (`sampev.onShowDialog`, строка ~5688): парсит диалог в `find_list`
  (ник, id, ранг, подразделение, телефон), открывает GUI `find_show`.
- **GUI** (строка ~6633): таблица сотрудников, выбор → действие.

### 8.5 Чёрный список (`/bladd`, `/bldel`, `/blcheck`, `/bllist` + GUI)

- **Файл:** `%LOCALAPPDATA%\helper_blacklist.json`.
- **Структура:** `blacklist_data[nick_lower] = { reason, added }`.
- **Функции:**
  - `addBlacklistNick(nick, reason)` (строка ~4612) — добавляет, дата `os.date('%d.%m.%Y')`.
  - `removeBlacklistNick(nick)` (строка ~4619) — удаляет.
  - `checkBlacklistNick(nick)` (строка ~4630) — проверяет наличие.
  - `loadBlacklist()` / `saveBlacklist()` (строки ~4590/~4603) — JSON загрузка/сохранение.
- **GUI** (строка ~6701): окно `blacklist_show` со списком и управлением.

### 8.6 Лифт ТВ-башни (`/tvlf`)

- **Синтаксис:** `/tvlf [этаж 1-21]`.
- **Логика** (`cmdTvlift`, строка ~4487): проверка позиции (зона ТВ-башни SF:
  `1839.5557, -1264.6527, 13.4299` → `1765.8602, -1319.9817, 134.1671`).
  РП-отыгровка (с учётом пола: «нажал»/«нажала») + `/tvlift`.

### 8.7 Увольнение (`/uninv`)

- **Синтаксис:** `/uninv [id] [причина]`.
- **Логика** (`cmdUninvite`, строка ~4511): проверка подключения игрока, РП-отыгровка
  (папка дел, подпись, `/r` сообщение, `/uninvite <id> <причина>`).
