# Быстрый старт: Запуск и расширение Helper Core

Этот гайд поможет вам установить скрипт, протестировать его в игре и понять, как добавлять новые функции в виде отдельных модулей.

---

## 1. Установка скрипта
1.  Установите чистую **GTA San Andreas**.
2.  Установите **SAMP 0.3.7-R1** или **0.3.7-R3** (в зависимости от вашей сборки).
3.  Установите мультиплеерный загрузчик **MoonLoader v0.26** (или новее).
4.  Скачайте библиотеку `mimgui` (обычно она идет в комплекте с MoonLoader, папка `lib` в директории `moonloader`).
5.  Поместите файл [helper_core.lua](file:///c:/Users/maxhy/OneDrive/Desktop/advance%20sapm/helper_core.lua) в папку `GTA San Andreas/moonloader/`.

---

## 2. Запуск в игре
1.  Зайдите на любой сервер (например, Advance RP).
2.  Скрипт автоматически загрузится. В чате появится сообщение:
    `Helper Core загружен. Открыть меню: F11 или /helper`
3.  Нажмите **F11** или введите команду `/helper` в чат.
4.  Перед вами откроется минималистичное окно с темной темой и списком модулей слева.

---

## 3. Как устроен код: Добавление нового модуля
Скрипт спроектирован так, чтобы вы могли легко добавлять неограниченное число новых функций, не ломая интерфейс. Всё, что нужно сделать — это добавить новый блок в таблицу `modules` внутри [helper_core.lua](file:///c:/Users/maxhy/OneDrive/Desktop/advance%20sapm/helper_core.lua#L26).

Каждый модуль выглядит следующим образом:

```lua
{
    id = "uniq_id",                          -- Уникальный текстовый идентификатор
    name = u8"🧊 Имя Модуля",                 -- Название (поддерживает эмодзи и русский текст через u8)
    description = u8"Описание модуля...",   -- Текст, объясняющий игроку работу модуля
    enabled = false,                         -- Состояние по умолчанию (включен/выключен)
    
    -- Рисует настройки в правой панели при активации
    drawSettings = function()
        imgui.Text(u8"Ваши настройки здесь:")
        -- Сюда можно добавлять чекбоксы, инпуты, слайдеры, кнопки
        if imgui.Button(u8"Нажми меня") then
            sampAddChatMessage("Кнопка нажата!", -1)
        end
    end,
    
    -- Вызывается при переключении чекбокса "Активировать модуль"
    onToggle = function(state)
        if state then
            -- Логика при включении (например, запуск потока или таймера)
        else
            -- Логика при выключении
        end
    end
}
```

## 4. Развитие до "Платформы" (Будущие шаги)

Когда базовый скрипт заработает в игре, мы сможем улучшить его по следующим направлениям:

### А. Динамическая загрузка из папки
Вместо того чтобы держать все модули в одном файле [helper_core.lua](file:///c:/Users/maxhy/OneDrive/Desktop/advance%20sapm/helper_core.lua), мы можем научить скрипт сканировать отдельную папку (например, `moonloader/helper_modules/`) и загружать каждый файл оттуда с помощью функции `require` или `loadfile`. Тогда добавление модуля будет сводиться к простому копированию файла в папку.

### Б. Связь с веб-сервером (Облако)
С помощью встроенных асинхронных HTTP-запросов (`copas.http` или `requests` в Lua) скрипт сможет:
*   Проверять лицензию пользователя.
*   Подгружать свежий список модулей с вашего сайта.
*   Отправлять статистику или сохранять настройки игрока в облаке.
